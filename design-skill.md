# Design Spec — Japandi 治愈风格 UI 规范

> 本文档是唯一的视觉规范来源。修改或生成任何 UI 时，必须遵循本规范。
> 风格关键词：Japandi / 日式治愈系 / 余白之美。整体气质是"安静、温暖、不打扰"。
> 核心原则：**不强调任何东西**。当你想加粗、加深、加大、加动效时，先问：不加行不行？

---

## 1. 颜色 Tokens

### 1.1 基础色（必须使用变量，禁止硬编码新色值）

| Token | 值 | 用途 |
|---|---|---|
| `--paper` | `#FAF7F2` | 页面底色（暖白/和纸色）。**禁止使用纯白 #FFFFFF 作为页面底色** |
| `--paper-deep` | `#F3F0E9` | 深一度的米色，用于区块分层、强调容器背景 |
| `--card` | `#FFFFFF` | 卡片背景。纯白只允许出现在卡片上，与 --paper 形成"两层纸"层次 |
| `--sand` | `#E8E2D6` | 图片占位、装饰色块 |
| `--line` | `#EDE8DF` | 分割线、边框。**禁止使用灰色 #E5E7EB 等冷灰边框** |

### 1.2 文字色

| Token | 值 | 用途 |
|---|---|---|
| `--ink` | `#3E3A36` | 主文字（墨色）。**禁止纯黑 #000000 和 #111827** |
| `--ink-soft` | `#8A857D` | 次级文字、描述 |
| `--ink-faint` | `#A8A296` | 辅助文字、placeholder、eyebrow 标签 |

### 1.3 主题色（低饱和，三色各司其职）

| Token | 值 | 深色配对 | 浅色配对（tint 背景） | 用途 |
|---|---|---|---|---|
| `--sage` | `#8FA98F` | `--sage-deep #6E8A6E` | `--sage-mist #EAF0E8` | 主色。主按钮、主要交互、品牌元素 |
| `--water` | `#6D8B90` | — | `--water-mist #E7EEEF` | 辅助色。次级分类、信息类标识 |
| `--clay` | `#B58A6A` | — | `--clay-mist #F4EBE3` | 点缀色。温度补偿、强调标记、暖色 badge |

**配色规则：**

- 彩色背景上的文字，必须使用同色系的深色（如 `--sage-mist` 背景配 `--sage-deep` 文字），禁止黑色/灰色文字压在彩色上
- 任何新增颜色必须"掺灰"：饱和度 ≤ 50%，禁止出现翠绿、正红、亮蓝等高饱和色
- 语义色替换：成功 → `--sage-deep`；警告 → `--clay`；信息 → `--water`；错误可用 `#C08585`（掺灰的红）
- 一个视图中主色（sage）实心填充最多出现 1 次（通常是主 CTA）

### 1.4 Tailwind 映射（shadcn 项目使用）

```js
// tailwind.config.js — theme.extend.colors
colors: {
  paper: { DEFAULT: '#FAF7F2', deep: '#F3F0E9' },
  ink: { DEFAULT: '#3E3A36', soft: '#8A857D', faint: '#A8A296' },
  sage: { DEFAULT: '#8FA98F', deep: '#6E8A6E', mist: '#EAF0E8' },
  water: { DEFAULT: '#6D8B90', mist: '#E7EEEF' },
  clay: { DEFAULT: '#B58A6A', mist: '#F4EBE3' },
  sand: '#E8E2D6',
  line: '#EDE8DF',
}
```

## 2. 字体 Typography

| 角色 | 字体 | 说明 |
|---|---|---|
| 标题（display） | `"Noto Serif SC", "Songti SC", serif` | 宋体/衬线。**标题必须用衬线体**，这是本风格的身份标识 |
| 正文/UI | `"Noto Sans SC", "PingFang SC", sans-serif` | 黑体 |
| 点缀（可选） | `"LXGW WenKai"` 霞鹜文楷 | 仅用于引言、语录等情绪化内容 |

**字号与字重：**

| 层级 | 字号 | 字重 | 行高 |
|---|---|---|---|
| H1 | 40–44px | **400（禁止加粗）** | 1.6 |
| H2 | 24–26px | 400 | 1.7 |
| H3 / 卡片标题 | 16–17px | 500 | 1.7 |
| 正文 | 14–15px | 400 | **2.0（硬性要求，不得低于 1.8）** |
| 辅助文字 | 12–13px | 400 | 1.9 |
| Eyebrow 标签 | 12–13px | 400 | 字间距 letter-spacing: 4–5px |

**字体规则：**

- 全局只允许两档字重：400 和 500。**禁止 600/700/bold**
- 大标题不加粗是本风格最重要的特征之一，靠字号和留白建立层级，不靠字重
- 禁止在段落中间加粗强调；需要强调时用 `--sage-deep` 颜色或单独成行

## 3. 间距与留白 Spacing

留白是本风格的核心。所有间距在常规设计系统基础上放大约 1.5 倍。

| 场景 | 值 |
|---|---|
| 卡片内边距 | ≥ 32px（大卡片 36–56px） |
| 区块之间垂直间距 | 88–96px |
| 卡片之间 gap | 16–20px |
| 段落间距 | ≥ 1.5em |
| 页面左右安全边距 | 32px（移动端 20px） |
| 内容最大宽度 | 1080px，正文段落 max-width ≤ 60ch |

**规则：** 信息密度宁低勿高。一屏（视口高度）内的独立信息模块不超过 3 个。当内容显得空，那是对的；当内容显得满，删内容而不是缩间距。

## 4. 形状 Shape

| 元素 | 圆角 |
|---|---|
| 按钮 | `999px`（全圆胶囊，本风格标配） |
| 卡片 | `18–20px` |
| 大容器/区块 | `24–26px` |
| 输入框 | `12px` |
| 图标底托 | 圆形 `50%` 或 `12px` |

**边框与阴影：**

- 边框：`0.5px 或 1px solid var(--line)`，或不用边框
- **禁止使用阴影做层级**。层级用背景色差实现（paper → paper-deep → card 三层）
- 允许的唯一阴影：交互聚焦时的 focus ring
- 禁止渐变、禁止玻璃拟态（backdrop-blur）、禁止 neon/glow 效果

---

## 5. 动效 Motion

| 参数 | 值 |
|---|---|
| 常规过渡时长 | `400–500ms`（比常规慢一倍，这是刻意的） |
| 入场动效时长 | `600–700ms` |
| 缓动函数 | `ease-out`（禁止 bounce、spring、overshoot） |
| 入场方式 | 轻微上浮 `translateY(14px) → 0` + 淡入 |
| Hover | 卡片 `translateY(-4px)`，按钮换色，均无阴影变化 |

**规则：**

- 动效语言只有一种：慢慢浮现。禁止弹跳、旋转、缩放脉冲、闪烁
- 必须支持 `@media (prefers-reduced-motion: reduce)`，降级为无动效
- 页面同屏同时运动的元素 ≤ 3 个

## 6. 组件规范

### 按钮

```
主按钮：bg --sage / 文字 --paper / 圆角 999px / padding 13px 32px / hover 变 --sage-deep
次按钮：bg --ink / 文字 --paper（深墨色胶囊，用于导航等）
文字按钮：--sage-deep 颜色 + "→" 后缀，无边框无背景
禁止：红/橙色大按钮、渐变按钮、带阴影按钮
```

### 卡片

```
bg --card / 圆角 18-20px / padding ≥32px / 无边框或 0.5px --line
结构：图标底托（44-48px 圆形，mist 色背景 + deep 色图标）→ 标题(500) → 描述(--ink-faint) → 文字链接
```

### 表单

```
输入框：bg #FFFFFF / 边框 1px --line / 圆角 12px / 高度 ≥44px / focus 边框变 --sage
label：13px --ink-soft，位于输入框上方 8px
placeholder：--ink-faint
错误提示：#C08585，文字提示不用红色边框全包围
```

### 标签 / Badge

```
胶囊形（999px）/ mist 色背景 + 同色系 deep 文字 / 12px / padding 4px 12px
例：bg --sage-mist + color --sage-deep
```

### 图标

```
线性图标（outline），线宽 1.5px，禁止实心（filled）图标
尺寸 16-22px，颜色跟随语境用 deep 系
也可用单个汉字做图标（如"息""行""坐"），衬线体，放在圆形 mist 底托上
```

### 空状态 / 加载

```
空状态：一句温和的话 + 淡色插画/图形，禁止"暂无数据"这种冷冰冰的文案
加载：缓慢的呼吸式透明度变化（opacity 0.4↔1，1.5s循环），禁止旋转 spinner
```

---

## 7. 文案语气（UI Copy）

- 短句，多用句号少用感叹号，全文最多一个"！"
- 温和不命令："开始一次练习" 而不是 "立即开始"；"慢慢来" 而不是 "快速上手"
- 空状态和错误给出路，不指责："还没有记录，从五分钟开始也可以。"
- 数字可以温度化："今日已有 128 人在练习" 而不是 "活跃用户 128"

---

## 8. 禁止清单（Hard Don'ts）

1. ❌ 纯白 `#FFFFFF` 页面底色 / 纯黑文字
2. ❌ 600/700 字重、大标题加粗
3. ❌ 高饱和色（翠绿、正蓝、亮橙、正红）
4. ❌ 渐变、阴影层级、玻璃拟态、glow
5. ❌ 弹跳/旋转/脉冲动效、<300ms 的生硬过渡
6. ❌ 行高 <1.8 的正文
7. ❌ 直角或小圆角（<12px）的卡片和按钮
8. ❌ 冷灰色系（#F3F4F6 / #6B7280 这类 Tailwind 默认灰）
9. ❌ 一屏塞入 4 个以上信息模块
10. ❌ 感叹号轰炸、命令式文案

---

## 9. 自检清单（改完 UI 后逐项核对）

- [ ] 页面底色是 `#FAF7F2`，卡片是纯白，形成两层纸
- [ ] 所有文字色来自 ink 三档，无纯黑
- [ ] 主色 sage 实心填充在当前视图只出现一次
- [ ] 标题为衬线体、400 字重
- [ ] 正文行高 ≥ 1.8
- [ ] 按钮为胶囊形，卡片圆角 ≥ 18px
- [ ] 无阴影、无渐变、无高饱和色
- [ ] 过渡动效 ≥ 400ms，ease-out，支持 reduced-motion
- [ ] 彩色背景上的文字为同色系深色
- [ ] 眯起眼看整个页面：应该是一片温暖的浅色，只有 1–2 个绿色焦点
